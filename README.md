# Spring Web MVC CRUD приложение для автоматизации планирования путешествий и управления транспортной инфраструктурой

Приложение позволяет эффективно управлять автобусами, городами и строить оптимальные маршруты между населенными пунктами 
с учетом различных критериев поиска.

## Технологический стек

- Java — основной язык разработки

- Spring Boot — создание микросервисной архитектуры

- Spring Data JPA — работа с реляционными базами данных

- Spring Web MVC — реализация REST API

- Hibernate — ORM для работы с БД

- Lombok — сокращение boilerplate кода

- Maven — управление зависимостями и сборка

  ## Функционал приложения

1. Управление автобусами
- CRUD операции для автобусов с валидацией входных данных

- Фильтрация автобусов по городам следования

- Возможность добавления/удаления городов в маршрут автобуса

- Каскадное обновление маршрутов при изменении данных

2. Управление городами
- Полноценный CRUD для городов с геолокационными координатами

- Поиск городов по автобусам, которые через них проходят

- Оптимизированные запросы для поиска по названиям

3. Поиск маршрутов
- Реализован алгоритм (на основе алгоритма Дейкстры) поиска маршрутов между городами с поддержкой:

Трех типов оптимизации:

- FASTEST — поиск самого быстрого маршрута (минимальное общее время)

- LESS_EXPENSIVE — поиск самого дешевого маршрута (минимальная стоимость)

- MIN_TRANSFERS — поиск маршрута с минимальным количеством пересадок

Динамическое построение маршрутов с учетом всех возможных пересадок между автобусами

Агрегация данных о стоимости, времени и количестве пересадок

## Архитектурные решения
- Многослойная архитектура: Controller → Service → Repository

- DTO паттерн для разделения слоев представления и бизнес-логики

- Кастомные исключения с глобальной обработкой ошибок через @ControllerAdvice

- Транзакционность на уровне сервисов для обеспечения целостности данных

## API Endpoints
- GET    /bus            - Получение списка автобусов с фильтрацией

- POST   /bus            - Создание нового автобуса

- GET    /bus/{id}       - Получение информации об автобусе

- PUT    /bus/{id}       - Обновление данных автобуса

- DELETE /bus/{id}       - Удаление автобуса

- GET    /bus/route/{id} - Получение маршрута автобуса

- PUT    /bus/addRoute/{id}   - Добавление городов в маршрут

- PUT    /bus/delRoute/{id}   - Удаление городов из маршрута


- GET    /city           - Получение списка городов

- POST   /city           - Создание города

- GET    /city/{id}      - Получение информации о городе

- PUT    /city/{id}      - Обновление данных города

- DELETE /city/{id}      - Удаление города


- GET    /route          - Поиск маршрутов между городами


## База данных (H2)

Спроектированные таблицы и связи:

- Таблица cities (id, name, latitude, longitude)
  
- Таблица buses (id, number, price)
  
- Таблица bus_city (связь многие-ко-многим с дополнительными полями)

## Особенности реализации 

- Валидация входных данных с помощью Bean Validation

- Обработка edge-cases:

- Поиск города, когда найдено несколько с одинаковым названием

- Ситуации, когда начальный и конечный город совпадают

- Обработка отсутствия маршрутов между городами

- Логирование всех операций для отладки и мониторинга

- RESTful принципы с правильными HTTP статусами
